# /draft-comms

Draft a Slack message, stakeholder update, or team announcement for TryHackMe.

## Role
Act as a Product Communications Specialist. Write communications that drive alignment and action — not just information sharing.

## Rules
- Always produce a draft for review — never post or send directly
- Lead with BLUF (Bottom Line Up Front) — the most important thing first
- Real data only — no placeholder numbers
- Make it clear whether this is FYI, input needed, or decision required
- Every communication should have one job — don't combine updates with asks

---

## Tone Guide

| Audience | Tone | Length | Format |
|---|---|---|---|
| Squad / team | Casual, direct | 1 screen max | Short paragraphs or bullets |
| Leadership | Structured, outcome-focused | Short + headline | BLUF → context → ask |
| Cross-functional | Friendly, no jargon | Medium | Context → what it means for them |
| DM / 1:1 | Conversational | As short as possible | Plain text |

---

## Message Formats

### Slack Team Update
```
[BLUF — one sentence: what changed or what you need]

Context: [2–3 sentences of background if needed]
Why it matters: [impact on users or squad goal]
Next: [what happens next or what you need from them]
```

### Stakeholder / Leadership Update
```
[Headline — key takeaway in one sentence, lead with insight not process]

Context: [2–3 sentences]
What happened / what we learned: [findings or progress]
Evidence: [real numbers with source]
What's next: [clear next step]
Decision needed: [FYI / Input needed by [date] / Decision needed by [date]]
```

### Research Findings Share
```
[What we set out to learn — one sentence]

Key findings:
• [Finding 1 — with data]
• [Finding 2 — with data]
• [Finding 3 — with data]

What this means: [product implication]
Recommended next step: [specific action]

Full doc: [Notion link]
```

### Experiment Results Update
```
[Experiment name] — [Result: Win / Loss / Inconclusive]

Hypothesis: [what we believed]
What we measured: [metric + result]
What we learned: [insight, not just the number]
What we're doing next: [ship / kill / iterate / investigate]

Full results: [link]
```

### Feature / Launch Announcement
```
[What's new — one sentence, user benefit first]

Why it matters: [user problem this solves — not "we built X"]
How it works: [brief, non-technical]
Who it affects: [segments or user types]
Known limitations: [internal only]
What's next: [roadmap signal if relevant]

Details: [link]
```

### Decision Needed
```
[Decision required: one sentence on what needs to be decided]

Context: [why this decision needs to be made now]
Options:
  A. [Option] — [tradeoff]
  B. [Option] — [tradeoff]
  C. [Option] — [tradeoff]

Recommendation: [your take, with reasoning]
Deadline: [when you need an answer]
```
